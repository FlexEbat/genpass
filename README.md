Утилита командной строки для безопасной генерации паролей. Не использует сетевые запросы, опирается на криптографически стойкий генератор псевдослучайных чисел (системный модуль `secrets`) и предоставляет расширенные возможности управления пулом символов, форматом вывода и буфером обмена.

---

## Требования

* Python 3.8+
* Поддерживаемые ОС: Linux / macOS / Windows
* Зависимости (опционально): библиотека `pyperclip` для более стабильной работы с буфером обмена.
* Системные пакеты (для Linux, если нет `pyperclip`): `xclip` или `xsel` для поддержки флага `--copy`.

---

## Основные принципы безопасности

* Исключено неявное сохранение паролей: данные выводятся только в `stdout` или в файл при явном указании флага.
* Работает полностью в оффлайн-режиме, исключая утечку через API.
* Применяется модуль `secrets`, разработанный специально для криптографических задач (в отличие от стандартного `random`).
* Присутствует механизм фоновой очистки буфера обмена и консоли для защиты от визуального перехвата (shoulder surfing).
* Программа не хранит историю генераций.

---

## Установка и запуск

Сделайте файл исполняемым (для UNIX-подобных систем):

```bash
chmod +x genpass.py
```

Запуск:

```bash
python3 genpass.py
```

При запуске без аргументов (или только с флагом `-ru`) скрипт автоматически переходит в интерактивный режим.

---

## Режимы работы

### Интерактивный режим (Меню)

Активируется при запуске без аргументов или с флагом `-i`. Пользователю предлагается выбор из 4 вариантов:

1. **Стандартный пароль:** классическая генерация с пошаговым выбором длины, регистров, цифр и спецсимволов.
2. **Парольная фраза (Diceware):** генерация читаемых фраз из словаря (например, `correct-horse-battery-staple`).
3. **Генерация по маске:** жестко заданный шаблон (например, 1 заглавная, 3 строчных, 4 цифры).
4. **Максимально защищенный:** автоматический пресет на 64 символа со всеми категориями, фоновой очисткой буфера и скрытием вывода.

### Конфигурационный файл

При первом запуске программа создает файл `config.json` в своей директории. В нем можно задать настройки по умолчанию (язык интерфейса, стандартную длину, путь к словарю и т.д.), чтобы не прописывать их каждый раз в консоли.

## Аргументы командной строки (CLI)

### Базовые параметры
```bash
-i, --interactive    # Запуск в интерактивном режиме
-ru, --ru            # Использовать русский язык интерфейса
--len N              # Длина стандартного пароля
--count N            # Количество генерируемых паролей
--preset NAME        # Использовать готовый пресет (web, strong, pin, wifi)
```

### Управление набором символов
```bash
-u, --upper          # Включить заглавные буквы (A-Z)
-l, --lower          # Включить строчные буквы (a-z)
-d, --digits         # Включить цифры (0-9)
-s, --symbols        # Включить спецсимволы
--safe               # Использовать только "безопасные" спецсимволы (без кавычек, слэшей и т.д.)
-A, --no-ambiguous   # Исключить визуально похожие символы (l, 1, I, o, O, 0)
--exclude "CHARS"    # Исключить конкретные символы из пула генерации
--custom "CHARS"     # Использовать исключительно переданный набор символов
```

### Парольные фразы (Diceware)
```bash
-w, --words N        # Количество слов в парольной фразе
--wordlist PATH      # Путь к текстовому файлу со словарем (по умолчанию: wordlist.txt)
--sep CHAR           # Разделитель между словами (по умолчанию: "-")
```
*Примечание: для режима Diceware требуется текстовый файл, содержащий слова (каждое с новой строки). Файл можно загрузить со сторонних ресурсов, например, weakpass.*

### Генерация по маске
```bash
--mask FORMAT        # Формат маски (U = A-Z, l = a-z, d = 0-9, S = Спецсимвол)
```
*Пример: `--mask "Ulll-dddd-S"` выдаст результат вида `Wxyz-1234-!`*

### Форматирование и вывод
```bash
--group N            # Разбить пароль на группы по N символов (разделение через --sep)
-q, --quiet          # Тихий режим (подавляет баннеры и энтропию, выводит только пароль)
-o, --output PATH    # Сохранить результат генерации в текстовый файл
--phonetic           # Вывести пароль с использованием фонетического алфавита (Alpha, Bravo...)
```

### Буфер обмена и безопасность экрана
```bash
-c, --copy           # Скопировать результат в буфер обмена
--clear N            # Таймер фоновой очистки буфера обмена (в секундах)
--clear-console      # Очистить консоль после истечения таймера (работает только вместе с --clear)
```

### Отладка
```bash
--selftest           # Запуск внутреннего тестирования генератора и пулов
```

## Оценка энтропии

Скрипт автоматически рассчитывает энтропию пароля и выводит оценку его криптостойкости по формуле: `log2(pool_size) * length` (для фраз: `words * log2(wordlist_size)`).

| Энтропия (бит) | Оценка |
| -------------- | ------ |
| ≥ 80           | STRONG |
| 60–79          | OK     |
| < 60           | WEAK   |

## Примеры использования

Генерация пароля на 24 символа без визуально похожих букв, с копированием в буфер:
```bash
python3 genpass.py --len 24 -ulds -A -c
```

Генерация парольной фразы из 5 слов с кастомным словарем:
```bash
python3 genpass.py -w 5 --sep "." --wordlist eff_large_wordlist.txt
```

Генерация по строгому шаблону с фоновой очисткой буфера через 10 секунд и последующей очисткой консоли:
```bash
python3 genpass.py --mask "Ulll-dddd-S" -c --clear 10 --clear-console
```

Генерация 100 пин-кодов и сохранение их в файл (в тихом режиме):
```bash
python3 genpass.py --preset pin --count 100 -o pins.txt -q
```

Фонетический вывод (удобно для диктовки пароля по телефону):
```bash
python3 genpass.py --preset pin --phonetic
```

---

## Коды возврата (Exit-codes)

* `0` — Успешное выполнение
* `1` — Ошибка переданных аргументов (неверный диапазон, отсутствие файла словаря и т.д.)
* `2` — Системная ошибка (например, отсутствие утилит для буфера обмена в ОС)
* `3` — Внутренняя самодиагностика (`--selftest`) не пройдена

---

## Примечания

**Хранение паролей:** 
Никогда не храните важные пароли в обычных текстовых файлах. Если используете флаг `--output`, позаботьтесь о последующем шифровании файла (GPG, 7zip) или импортируйте его в менеджер паролей.

**Почему скрипт просит словарь локально:**
Скачивание словарей из интернета в момент генерации нарушает концепцию offline-утилиты и создает риск MITM-атак. Загружайте проверенные словари самостоятельно. 

**Совместимость маски и флагов исключения:**
Если вы используете маску (`--mask`) совместно с флагами `--exclude` или `-A`, скрипт корректно отфильтрует пулы для каждого символа маски. Если пул окажется пустым, программа сообщит об ошибке и безопасно завершит работу.
